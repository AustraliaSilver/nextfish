Ý tưởng: Cải tiến Reverse Futility Pruning (RFP) cho Nextfish
Dựa trên: PlentyChess
Mục tiêu: Loại bỏ các node ở độ sâu thấp nếu điểm số đánh giá tĩnh (Static Evaluation) vượt quá một ngưỡng an toàn, giúp tiết kiệm tài nguyên tính toán.

Chi tiết thay đổi:
1. RFP (còn gọi là Static Null Move Pruning) sẽ được áp dụng mạnh mẽ hơn ở các node gần lá.
2. Ngưỡng (Margin) sẽ được tính toán động dựa trên độ sâu: margin = depth * 75.
3. Nếu (staticEval - margin >= beta), chúng ta có thể trả về điểm số sớm.

Công thức đề xuất:
if (depth <= 8 
    && !isCheck 
    && !rootNode 
    && staticEval - (rfpMargin(depth)) >= beta) {
    return staticEval - (rfpMargin(depth));
}

Trong đó rfpMargin(depth) = depth * 75 - (improving ? 50 : 0);
